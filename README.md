# 학습자료 기반 스터디 챗봇 (가칭)

> 내가 올린 강의자료·PDF를 학습해서 질문에 답하고, 퀴즈를 자동 생성해주는 RAG 기반 스터디 챗봇 서비스.

개인 학습자료를 업로드하면 그 내용을 근거로 답변하는 챗봇입니다. 일반 AI 챗봇과 달리, **내 자료에 있는 내용만 근거로** 답하므로 정확하고, 답변의 출처도 함께 보여줍니다.

---

## 핵심 기능

- **자료 기반 Q&A (RAG)** — 업로드한 자료에서 관련 내용을 검색해, 그것을 근거로 답변 생성
- **실시간 스트리밍 (WebSocket)** — 답변이 한 글자씩 실시간으로 표시
- **퀴즈 자동 생성** — 자료를 바탕으로 퀴즈를 만들고 채점·오답 기록
- **JWT 인증** — 자체 회원가입/로그인 + 소셜 로그인(구글·카카오)

---

## 기술 스택

| 구분 | 기술 |
|------|------|
| 백엔드 | Spring Boot 3.x, Spring AI 1.x, Java 21 |
| 프론트엔드 | React |
| 데이터베이스 | PostgreSQL (일반 데이터), Redis (벡터·세션·캐시) |
| 인증 | JWT + 리프레시 토큰, OAuth 2.0 |
| AI | RAG (Retrieval-Augmented Generation), LLM API |
| 인프라 | Docker, (배포: Vercel + 백엔드 호스팅) |

## 아키텍처 개요

```
[사용자 브라우저 / React]
          |  (HTTP + WebSocket)
          v
[Spring Boot 서버] ── 모든 로직의 중심
   |          |            |
   v          v            v
[PostgreSQL] [Redis]   [LLM API]
 일반 데이터  벡터·세션·캐시  임베딩·답변 생성
```

모든 요청은 Spring Boot 서버를 거친다. 브라우저가 DB나 외부 AI에 직접 접근하지 않으며, 인증·검증·비즈니스 로직이 서버에 집중된다.

### 핵심 동작 흐름 (RAG)

1. 사용자가 질문
2. 서버가 질문 접수 (인증·권한 확인)
3. 관련 자료 검색 (Redis 벡터 검색)
4. 질문 + 찾은 자료를 함께 묶기
5. LLM API에 전달 → 자료 기반 답변 생성
6. WebSocket으로 실시간 스트리밍
7. 사용자 화면에 답변 도착

## 개발 단계 (워킹 스켈레톤)

기능을 한 번에 다 만들지 않고, 작동하는 최소 단위부터 단계적으로 확장한다.

- **1단계 (뼈대)**: 자체 로그인 → 자료 업로드 → 단순 Q&A 1회 관통
- **2단계 (핵심)**: PDF 추출, WebSocket 스트리밍, 리프레시 토큰, 처리 상태 표시
- **3단계 (확장)**: 퀴즈, 캐싱, rate limiting, 대화 기록, 소셜 로그인
